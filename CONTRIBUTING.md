# Contributing

You do not need permission and you do not need to be an expert. You need to be specific.

## Three ways in

**Discussions** — thinking aloud, questions, disagreement, "has anyone considered". Nothing here changes the files. Start here if you are unsure.

**Issues** — a specific proposal: a page that is wrong, a missing concept, an objection that is not answered honestly, a source that would strengthen a claim. One issue per idea.

**Pull requests** — an actual change to the text. Fix a sentence or write a whole page. Small ones are welcome; typos and clearer wording are real contributions.

## How to make a change

Every page is a plain text file in Markdown. On GitHub you can edit one in your browser: open the file, click the pencil, make the change, and describe it in a sentence at the bottom. That creates a pull request. No software to install.

## What makes a contribution likely to be accepted

**Follows the schema.** Every concept page uses the same structure. See `SCHEMA.md`. A page that skips the objections section will be sent back for them.

**Plain language.** If a fifteen-year-old cannot follow it, rewrite it. No jargon that is not immediately explained. Short sentences beat clever ones.

**Specific.** "Communities could share resources" is not a contribution. "Here is how surplus is matched to need, here is what happens when two communities need the same thing, here is what is unresolved" is.

**Honest about limits.** Every page has an unsolved section and an objections section. Filling them in is more valuable than adding another paragraph of what works. Contributions that only strengthen the case tend to be weaker than ones that stress-test it.

**Evidenced where possible.** Existing precedents, real projects, published data. Where there is no evidence, say so plainly rather than implying there is.

## What will not be accepted

Party politics, or attacks on named individuals or organisations. Predictions of collapse used as an argument. Anything that reintroduces price, profit or ownership as the mechanism, in any form, however kind. Content lifted from copyrighted sources without permission. Abuse of other contributors.

## A note on disagreement

Some disagreements are about facts and can be settled. Some are about values and cannot. The second kind are not a problem to be eliminated. Record them honestly on the page, and if you believe the project has taken a wrong turn, fork it. That is a legitimate and welcome outcome, not a failure.
